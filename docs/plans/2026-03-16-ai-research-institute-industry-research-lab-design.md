# AI研究院 - 产业研究室设计文档

**日期**: 2026-03-16  
**状态**: 已确认设计  
**版本**: 1.0  
**定位**: 面向专业投研用户的双引擎 AI 产业研究操作系统

## 1. 产品定义

`AI研究院 > 产业研究室` 不是一个“帮用户生成图谱”的功能页，而是一个围绕产业和个股研究构建的专业工作台。产品核心是把研究行为拆成两套连续协同的引擎：

- `深度研究引擎`: 用于首次看懂一个行业、主题或个股，建立产业链认知、核心关系和可验证结论。
- `持续覆盖引擎`: 用于在已有研究资产基础上持续跟踪变化，识别是否有足够强的新证据改变图谱和结论。

两套引擎共享同一套研究资产底座：

- 研究对象
- 实体库
- 关系图谱库
- 证据库
- 研究结论库
- 版本历史
- 监控规则
- 预警事件

产品目标不是替代分析师，而是让分析师拥有一支结构化、透明、可审计的 AI 研究团队。

## 2. 目标用户与设计原则

### 2.1 目标用户

核心用户为专业投研用户，包括但不限于：

- 二级市场行业研究员
- 个股分析师
- 投资经理
- 产业研究团队
- 高信息密度的专业投资者

### 2.2 设计原则

产品体验必须围绕以下原则展开：

1. `高信息密度`
   页面默认不是空白引导，而是研究对象、变化、证据、图谱、任务状态同时可见。
2. `强控制感`
   用户可以指定研究范围、信息源、关系粒度、输出结构和优先级，而不是只能接受一键生成。
3. `证据优先`
   无证据不入图，有冲突不隐藏，有更新必留痕。
4. `持续性`
   一次研究必须沉淀为可复用资产，后续更新必须基于已有图谱和结论演化。
5. `专业工作流优先`
   产品心智应接近研究终端，不应退化成通用聊天页。
6. `键盘优先`
   高频操作应支持搜索、命令面板、快速跳转、筛选和批量操作。

## 3. 双引擎产品骨架

### 3.1 顶层结构

建议在现有产品中新增：

- 一级菜单: `AI研究院`
- 二级菜单: `产业研究室`

进入《产业研究室》后，顶部只保留两个主模式切换：

- `深度研究`
- `持续覆盖`

同时提供四个共享资产中心：

- `图谱库`
- `公司/行业档案`
- `证据库`
- `规则中心`

再加一个总入口页：

- `研究总览`

### 3.2 研究总览

研究总览不是介绍页，而是专业用户每日进入后的默认控制面板。建议展示：

- 今日变化
- 高优先级预警
- 最近打开的行业/个股
- 覆盖池健康度
- Agent 运行队列
- 待裁决争议项
- 最近版本变更

它承担“今天先看什么、先做什么”的入口角色。

## 4. 深度研究引擎

### 4.1 目标

帮助用户在 30 到 90 分钟内完成一次高质量、可追溯、可编辑的行业或个股建模，产出：

- 可编辑上下游图谱
- 研究摘要
- 关键争议点
- 风险与替代路径
- 待跟踪清单

### 4.2 启动方式

深度研究引擎首页只要求用户提供最少但最关键的输入：

- 研究对象: 行业 / 主题 / 个股
- 研究深度: 快速、标准、深度
- 输出目标: 上下游图谱 / 价值链拆解 / 公司穿透 / 竞争格局

可选高级控制：

- 市场范围: A股 / 港股 / 美股 / 全球
- 信息源策略: 一级信源优先 / 全量搜索 / 排除低可信来源
- 结构偏好: 行业全景 / 核心公司中心 / 区域链路

### 4.3 页面结构

建议采用三栏主结构：

- 左栏 `研究编排区`
  - 阶段列表
  - Agent 任务泳道
  - 优先级
  - 运行状态
  - 异常提示
- 中栏 `主工作画布`
  - 默认动态图谱
  - 可切换价值链视图、表格视图、事件视图、公司卡片视图
- 右栏 `证据检查器`
  - 来源链接
  - 摘录片段
  - 抓取时间
  - 置信度
  - 冲突证据
  - Agent 判断依据

### 4.4 关键交互

用户应能够在研究过程中持续介入，而不是只能等待结果：

- 只保留 A 股上市公司
- 把某公司设为核心节点重新展开
- 排除低可信来源
- 分离海外供应链
- 只看一级供应商
- 对单条关系发起重验证
- 冻结当前结构后继续补证

系统必须将这些控制转化为显式任务，而不是隐藏地重新生成整个结果。

### 4.5 输出结果

一次深度研究结束后，结果不应只有一份报告，而应形成并列资产：

- 图谱版本
- 结构化结论
- 争议项清单
- 核心证据集合
- 后续覆盖规则建议

这些资产自动进入持续覆盖引擎。

## 5. 持续覆盖引擎

### 5.1 目标

持续覆盖引擎负责维护已有研究对象的新鲜度。它不只是资讯推送，而是判断新信息是否足以改变既有图谱和结论。

### 5.2 页面结构

建议采用变化优先的监控终端布局：

- 左栏 `覆盖列表`
  - 行业
  - 个股
  - 主题篮子
  - 收藏对象
  - 自定义标签分组
- 中栏 `今日变化流`
  - 新增关系
  - 核心公司事件
  - 价格/产能/订单变化
  - 政策事件
  - 海外替代进展
  - 观点分歧
- 右栏 `影响分析面板`
  - 影响的节点和边
  - 影响的研究结论
  - 建议动作
  - 版本 diff 入口

### 5.3 核心机制

每条进入系统的新证据都应该经过三层判断：

1. `是否可信`
2. `是否与既有图谱或结论相关`
3. `是否足以改变认知`

系统输出的动作建议应分层：

- `建议更新图谱`
- `建议复核`
- `仅存档`
- `加入争议观察`

### 5.4 专业价值

持续覆盖引擎的专业价值体现在：

- 用户优先看到“认知变化”，而不是噪音信息
- 所有变化都可追溯到图谱和结论
- 所有重要更新都生成版本历史
- 用户可以从任何一次变化回跳到对应的证据、关系和原始研究

## 6. 多角色 Agent 协作

### 6.1 角色设计

建议将 AI 研究团队显式拆成以下角色：

- `首席分析师`
  - 负责拆解研究任务
  - 决定研究路径
  - 汇总阶段结论
- `产业链侦察员`
  - 负责网页搜索
  - 调用 `fetch_url`
  - 抓取公司、行业和政策资料
- `公司关系分析师`
  - 抽取供应商、客户、竞品、替代关系
  - 识别上下游层级
- `证据审计官`
  - 做交叉验证
  - 标注冲突
  - 输出置信度
- `图谱编辑师`
  - 实体去重
  - 关系归类
  - 维护图谱结构
- `投资秘书`
  - 生成摘要
  - 提炼关键结论
  - 产出待跟踪清单

### 6.2 协作方式

页面顶部建议保留一个 `研究指挥栏`，支持用户对整个 Agent 团队下达高精度命令：

- 重跑上游验证
- 锁定当前图谱
- 只保留一级信源
- 将某公司加入重点覆盖
- 将争议项提升优先级

同时，在主界面展示 `Agent 协作泳道`，让用户看到：

- 哪个 Agent 正在工作
- 当前在用什么工具
- 进度到哪一步
- 是否遇到冲突
- 是否需要人工裁决

## 7. 世界级交互设计细节

### 7.1 全局搜索与命令面板

产品应提供研究级全局搜索，支持键盘快捷键 `/` 或 `Cmd/Ctrl + K`。

搜索范围包括：

- 行业
- 个股
- 图谱
- 证据片段
- 实体
- 关系边
- 结论
- 监控规则

命令范围包括：

- 展开某节点上下游
- 比较某两个版本
- 只显示某关系类型
- 导出表格
- 发起深度研究
- 加入持续覆盖

### 7.2 图谱交互

图谱必须是主工作层，而不是展示层。建议支持：

- 按关系类型过滤
- 按置信度着色
- 按时间切片
- 锁定核心节点
- 批量展开二级链路
- 表格/图谱双向切换
- 悬停预览证据
- 双击进入公司穿透页

### 7.3 版本系统

版本系统是长期壁垒。每次深度研究、每次覆盖更新，都应生成一个图谱版本和结论版本。用户应能查看：

- 图谱 diff
- 结论 diff
- 新增关系
- 失效关系
- 证据强度变化
- 风险级别变化

用户还应能将版本标记为：

- 投委会版
- 内部基线版
- 草稿版

### 7.4 批注与裁决

专业研究过程需要保留争议和假设。建议支持：

- 节点批注
- 关系批注
- 假设标记
- 待验证标记
- 高争议标记
- 人工裁决记录

## 9. 数据模型建议

建议围绕以下核心模型设计：

  - 类型: 行业 / 个股 / 主题
  - 基础属性
  - 当前状态
  - 所属覆盖池
- `Entity`
  - 公司
  - 产品
  - 技术
  - 政策
  - 地区
- `Relationship`
  - 上游
  - 下游
  - 客户
  - 供应商
  - 替代
  - 竞争
  - 影响
- `Evidence`
  - 来源
  - 摘录
  - 时间
  - 可信度
  - 抽取方式

  - 结论正文

  - 风险点

  - 建议动作

## 10. 可信机制与错误处理

对专业投研用户，可信度比炫技更重要。系统必须显式呈现不确定性。

### 10.1 基本原则

- 无证据不入图
- 有冲突不隐藏
- 低置信度不伪装成确定结论
- 重要更新必须生成版本
- 失败状态必须可见

### 10.2 状态定义

建议统一以下状态：

- `已验证`
- `待验证`
- `低置信`
- `冲突中`
- `抓取失败`
- `已归档`

### 10.3 异常场景

重点处理以下问题：

- 网页抓取失败
- 来源内容为空或失真
- 同一实体重复归并
- 相互矛盾的上下游关系
- 图谱更新覆盖旧结论
- Agent 工具调用中断

系统应在界面中直接暴露这些问题，而不是静默失败。

## 12. 范围分层

### 12.1 MVP

第一阶段只打通主闭环：

- 行业/个股研究启动器
- 多 Agent 研究过程可视化
- 上下游图谱自动生成
- 节点/关系证据查看
- 图谱保存与历史打开
- 基础持续覆盖更新

### 12.2 Pro

第二阶段增强专业能力：

- 图谱 diff
- 结论 diff
- 置信度分层
- 覆盖池
- 监控规则编排
- 结构化导出
- 批注与裁决
- 命令面板

### 12.3 旗舰版

第三阶段建立壁垒：

- 跨行业联动图谱
- 时间轴回放
- 主题篮子覆盖
- 团队协作
- 研究分支
- 投委会快照
- 自动周报 / 月报
- 异常变化检测

## 13. 成功指标

建议围绕以下指标衡量产品是否成立：

- 首次研究完成率
- 单次深度研究平均时长
- 图谱保存率
- 已保存对象二次打开率
- 持续覆盖对象留存率
- 证据点击率
- 版本 diff 使用率
- 争议项裁决率
- 监控预警采纳率

## 15. 非目标

第一版不建议把范围扩展到以下方向：

- 通用投研聊天助手
- 大而全的社交协作
- 全平台资讯聚合器
- 与交易执行系统强绑定
- 面向泛用户的轻内容社区

## 16. 结论

《产业研究室》的正确产品形态，是一个面向专业投研用户的双引擎研究操作系统，而不是单次生成工具。其最核心的护城河不是图谱可视化本身，而是：

- 证据驱动
- 版本可回放
- 深研与覆盖连续贯通
- Agent 协作透明可控

如果这四件事做透，它会成为投研流程中的基础设施；如果做不透，它只会是一个看起来很强的 AI 演示页。
