## 1. 项目架构

### 1.1 技术栈选择
- **前端**：React + TypeScript + Ant Design + Lucide React (图标库)
- **后端**：Python + FastAPI
- **数据库**：SQLite（开发环境）/ PostgreSQL（生产环境）
- **存储**：本地文件系统（用于临时存储上传文件）

## 2. 核心功能实现

### 2.1 前端实现

#### 2.1.1 设计系统（参考提供的风格）
- **颜色方案**：使用白色为主背景，浅蓝色为强调色，辅以浅灰色
- **排版**：使用现代无衬线字体，建立清晰的字体层次结构
- **间距**：采用 8px 基准的间距系统，确保界面元素布局合理
- **圆角**：统一使用 8-12px 的圆角，使界面更加柔和
- **阴影**：使用轻微的阴影效果，增强界面层次感

#### 2.1.2 布局组件
- **AppLayout**：实现整体布局，包含侧边菜单和内容区域，顶部添加"+ 新解析"按钮
- **SideMenu**：实现侧边栏，包含"任务管理"等菜单项，保持简洁的设计风格

#### 2.1.3 文档解析页面
- **TaskList**：左侧任务列表，支持新建任务，显示任务状态，设计与参考页面一致
- **FileUpload**：右侧上传区域，支持拖拽上传和点击上传，包含"上传文件"和"网页链接"选项
- **FilePreview**：上传后左侧预览区域，显示文件基本信息和预览
- **MarkdownViewer**：上传后右侧区域，显示解析出的 Markdown 内容（包含图片）
- **示例区域**：添加示例文档展示区域，与参考页面风格一致

#### 2.1.4 API 服务
- 封装前端与后端的通信接口
- 处理文件上传、任务创建、任务状态查询等

### 2.2 后端实现

#### 2.2.1 数据库配置
- 使用 SQLAlchemy ORM
- 配置 SQLite 或 PostgreSQL 连接
- 实现数据库迁移

#### 2.2.2 数据模型
- **Task**：任务模型，包含任务 ID、文件名称、状态、创建时间、更新时间等字段

#### 2.2.3 路由
- `/api/document/upload`：处理文件上传
- `/api/document/parse`：创建解析任务
- `/api/document/task/{task_id}`：查询任务状态
- `/api/document/tasks`：获取任务列表

#### 2.2.4 minerU API 服务
- 封装 minerU API 调用
- 处理 Token 认证
- 实现文件上传、任务创建、状态查询等功能

#### 2.2.5 文件处理
- 临时存储上传的文件
- 处理文件格式验证
- 管理文件上传和删除

## 3. 关键流程

### 3.1 文件上传解析流程
1. 用户在前端上传文件
2. 前端将文件发送到后端 `/api/document/upload` 接口
3. 后端接收文件并存储到临时目录
4. 后端调用 minerU API 申请文件上传链接
5. 后端将文件上传到 minerU 提供的链接
6. minerU 自动创建解析任务并开始解析
7. 后端创建数据库任务记录并更新状态
8. 前端轮询后端 `/api/document/task/{task_id}` 接口获取解析状态
9. 解析完成后，后端从 minerU 下载解析结果（包含 Markdown 文件）
10. 后端更新数据库任务状态
11. 前端显示解析结果

### 3.2 任务管理流程
1. 用户在前端创建新任务
2. 前端发送请求到后端 `/api/document/tasks` 接口
3. 后端创建任务记录到数据库并返回任务 ID
4. 前端使用任务 ID 进行文件上传和解析
5. 后端更新任务状态到数据库
6. 前端从数据库获取任务列表和状态
